# Notes Standard

本文件定义笔记生成的统一规范，供 Codex 等代理遵循。

## 标题格式

- 每篇第一行固定为：`# <编号>. <词1>，<词2>，<词3>`
- 三个词任意，中英均可，用中文逗号分隔。
- 示例：`# 20. Rooms，Areas，Flow`

## 段落规则

- 日期与碎碎念放在标题下第一段，不另起更高层级标题。
- 全篇只用二级标题 `##`，不出现 `###` 或更高层级标题。
- 不强制固定的 `##` 结构，按内容自然拆分即可。
- 每个 `##` 后先写一段人话解释，再给代码块。

## 代码规则

- 代码示例使用 `lua` 代码块。
- 代码块偏多一些，以确保易懂。
- 单段代码不宜过长，可使用 `:::code-group :::` 分组。
- 尽量交代“为何这样做”与“何时不适用”。
- 代码内命名统一使用 snake_case，文件命名也用 snake_case。

## Exercise 格式

- 每个 Exercise 标题后先放题目原文引用块（`>`），再写解释段。
- 纯解析题只写文字，不写代码。
- 需要代码时，解释段在前，代码块在后；代码块可用 `:::code-group :::` 分组。
- 代码与文件名都用 snake_case。

## 风格与引用

- 主基调必须是“克制的讲义式”：人话清楚、举例多、代码多，但语言克制、言简意赅。
- 避免说明书式的生硬流程感，也避免日记式的主观叙述。
- 句子短、节奏快，像课堂板书但逻辑清楚。
- 如有助理解，优先引用游戏例子；理论框架仅在帮助理解时使用。

## 风格清单（更克制）

- 句子短，少形容，多动作，留白多。
- 叙事像聊天，但逻辑清楚，不卖弄。
- 细节具体，点到为止，不堆砌。
- 情绪克制，不煽情，以事实与观察留余味。
- 比喻与引用少而准，一次即可，不连发。
- 允许轻微自嘲或幽默，但不刻意。
- 先人话后术语，术语只保留必要部分。
