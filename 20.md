## Room 是啥

这篇讲的就是“先把舞台搭好再写剧情”。Room 是舞台（场景/界面），Area 是舞台上的对象管理员。玩法还没开始写，但这两块先设计好，后面就不会一团糟。

先把 Room 的形态记下来，它其实就是普通类：

```lua
Stage = Object:extend()

function Stage:new()
    -- 这里只放这个 Room 自己需要的东西
end

function Stage:update(dt)
end

function Stage:draw()
end
```

## 最简切换

最简单的 Room 系统只需要一个 `current_room` 和一个切换函数。游戏每一帧只对当前 Room 做更新和绘制。

```lua
function love.load()
    current_room = nil
end

function love.update(dt)
    if current_room then current_room:update(dt) end
end

function love.draw()
    if current_room then current_room:draw() end
end

function gotoRoom(room_type, ...)
    current_room = _G[room_type](...)
end
```

`room_type` 是字符串，比如 `'Stage'`，在 Lua 里全局表 `_G` 可以用字符串取到类。  
所以 `gotoRoom('Stage')` 实际就是 `current_room = Stage(...)`。每次切换都会新建一个 Room，旧的没人引用就会被 GC 回收。  
这很“爽快”：代码短、脑子负担小；代价是不能保留上次的状态。

## 什么时候拆 Room

用游戏来想会更清楚。Nuclear Throne 那种流程：主菜单 -> 选角 -> 游戏 -> 选被动 -> 再回游戏。  
看似画面很多，但“加载界面”“死亡界面”更像是在游戏画面上盖了一层 UI，不一定值得拆成 Room。  
经验就是：逻辑完全不同就拆 Room；只是同一套逻辑里的临时状态，就留在同一个 Room 里处理。

## 持久 Room

更复杂的游戏会需要“持久 Room”，也就是切走也不销毁，回来的时候状态还在。

```lua
function love.load()
    rooms = {}
    current_room = nil
end

function addRoom(room_type, room_name, ...)
    local room = _G[room_type](room_name, ...)
    rooms[room_name] = room
    return room
end

function gotoRoom(room_type, room_name, ...)
    if current_room and rooms[room_name] then
        if current_room.deactivate then current_room:deactivate() end
        current_room = rooms[room_name]
        if current_room.activate then current_room:activate() end
    else
        current_room = addRoom(room_type, room_name, ...)
    end
end
```

这个版本多了 `activate/deactivate`，可以用来存档、读档、清引用。  
适合像《以撒》这种“能回头走”的结构：房间被打过、石头被炸掉，回去时还要保持当时的状态。

## 转场限制

如果你要做房间之间的过渡动画，可能要“同时画两个 Room”，单一 `current_room` 会不够，这里只是简化版，先记住这个限制。

## Area 的职责

Room 是导演，Area 是管场。它负责把“所有对象”统一更新、绘制、移除。

```lua
Area = Object:extend()

function Area:new(room)
    self.room = room
    self.game_objects = {}
end

function Area:update(dt)
    for i = #self.game_objects, 1, -1 do
        local obj = self.game_objects[i]
        obj:update(dt)
        if obj.dead then table.remove(self.game_objects, i) end
    end
end

function Area:draw()
    for _, obj in ipairs(self.game_objects) do obj:draw() end
end
```

## GameObject 基类

所有对象继承自一个基类 `GameObject`，这样能保证每个对象都有位置、ID、timer 之类的通用属性：

```lua
GameObject = Object:extend()

function GameObject:new(area, x, y, opts)
    local opts = opts or {}
    if opts then for k, v in pairs(opts) do self[k] = v end end

    self.area = area
    self.x, self.y = x, y
    self.id = UUID()
    self.dead = false
    self.timer = Timer()
end

function GameObject:update(dt)
    if self.timer then self.timer:update(dt) end
end
```

`opts` 是个小技巧：创建对象时可以顺手塞一堆参数，直接挂到对象身上。

## UUID 与随机

ID 需要唯一，所以用 UUID：

```lua
function UUID()
    local fn = function(x)
        local r = love.math.random(16) - 1
        r = (x == "x") and (r + 1) or (r % 4) + 9
        return ("0123456789abcdef"):sub(r, r)
    end
    return (("xxxxxxxx-xxxx-4xxx-yxxx-xxxxxxxxxxxx"):gsub("[xy]", fn))
end
```

用 `love.math.random` 是为了吃到 LÖVE 自己的随机种子初始化，不然每次启动可能都生成同一批 ID。

## 往 Area 里塞对象

最后把对象丢进 Area：

```lua
function Area:addGameObject(game_object_type, x, y, opts)
    local game_object = _G[game_object_type](self, x or 0, y or 0, opts or {})
    table.insert(self.game_objects, game_object)
    return game_object
end
```

## Room 里怎么用 Area

一个最朴素的 Room 里会这样用：

```lua
function Stage:new()
    self.area = Area(self)
    self.area:addGameObject('Player', 100, 120)
end

function Stage:update(dt)
    self.area:update(dt)
end

function Stage:draw()
    self.area:draw()
end
```
